# 个股行情页订单与持仓查询功能设计文档

> **版本**：v1.0  
> **日期**：2026-05-07  
> **状态**：待评审  
> **关联模块**：行情模块 / 交易模块  

---

## 1. 背景与目标

### 1.1 背景
当前 App 的持仓与订单信息仅在「交易」Tab 中以账户视角聚合展示。用户在查看某只个股行情时，若需确认自身持仓或未完成订单，必须切换页面，路径割裂，影响交易决策效率。

### 1.2 目标
在个股行情页内，提供与该股票相关的**持仓快照**与**未完成订单**查询能力，使用户在盯盘时能够实时感知自身仓位与委托状态，缩短决策路径，提升交易转化率。

### 1.3 成功指标
- 用户从行情页发起交易的比例提升（衡量持仓/订单信息对交易决策的促进作用）
- 行情页至交易 Tab 的反复跳转次数下降

---

## 2. 范围

### 2.1 In Scope
- 个股行情页底部「快览条」展示
- 底部快览条点击/上滑后的「半屏抽屉」展示
- 半屏抽屉内持仓详情与未完成订单列表
- 持仓 Tab 快捷买入/卖出入口
- 订单 Tab 快捷撤单、改单跳转

### 2.2 Out of Scope
- 跨市场关联展示（如 00700 与 TCEHY）
- 多账户持仓聚合与切换
- 交易规则差异的特殊处理（竞价、盘前盘后等）
- 货币符号的差异化展示
- 历史成交记录的完整列表（仅展示未完成订单）

## 4. 数据字段

### 4.1 持仓侧（个股维度）

| 字段 | 说明 | 示例 |
|------|------|------|
| 持仓数量 | 当前持有总股数 | 100 |
| 可卖数量 | 当前可卖出股数（考虑 T+1/T+2 冻结） | 100 |
| 成本价 | 持仓成本均价 | 380.00 |
| 当前市值 | 最新价 × 持仓数量 | 67,000.00 |
| 当日盈亏金额 | 持仓数量 × (最新价 - 昨收价) | +1,234.50 |
| 当日盈亏百分比 | 当日盈亏金额 / (持仓数量 × 昨收价) | +1.87% |
| 持仓盈亏金额 | 持仓数量 × (最新价 - 成本价) | -16,939.25 |
| 持仓盈亏百分比 | 持仓盈亏金额 / (持仓数量 × 成本价) | -6.64% |
| 持仓占比 | 该股市值 / 账户总资产市值 | 11.27% |

### 4.2 订单侧（该股未完成订单）

| 字段 | 说明 | 示例 |
|------|------|------|
| 买卖方向 | 买入 / 卖出 | 买入 |
| 订单状态 | 待成交 / 部分成交 | 部分成交 |
| 委托价格 | 用户委托的限价 | 375.00 |
| 委托数量 | 下单时的委托股数 | 100 |
| 成交数量 | 已成交股数 | 30 |
| 订单类型 | 限价 / 市价 / 条件单 | 限价 |
| 下单时间 | 订单提交时间戳 | 09:42:15 |
| 订单 ID | 全局唯一订单标识 | 202605070001 |

## 5. 设计方案

### 5.1 总体架构

采用 **「底部快览条 + 半屏抽屉」** 双层架构：

```
个股行情页
├── 头部信息区（股票名称、代码、实时价格）
├── K线/分时图区
├── 盘口/买卖档区
├── 资讯/数据Tab区
└── 底部快览条（Fixed Bottom）
    ├── 状态A：有持仓 → 展示持仓数量 + 当日盈亏
    ├── 状态B：无持仓但有未完成订单 → 展示进行中订单数量
    └── 点击/上滑 → 拉起半屏抽屉
        ├── Tab 1: 持仓（该股完整持仓信息）
        └── Tab 2: 订单（该股未完成订单列表，最多展示最新10条）
```

### 5.2 底部快览条（Quote Footer Bar）

#### 5.2.1 展示规则

| 用户状态 | 展示内容 | 可见性 |
|----------|----------|--------|
| 未登录 | 隐藏快览条 | 不可见 |
| 已登录，无持仓且无未完成订单 | 隐藏快览条 | 不可见 |
| 已登录，有持仓 | 左侧：「持仓 {数量} 股」<br>右侧：「{当日盈亏金额} ({当日盈亏百分比})」 | 可见 |
| 已登录，无持仓但有未完成订单 | 左侧：「进行中订单」<br>右侧：「{N} 笔」 | 可见 |
| 已登录，既有持仓又有未完成订单 | 优先展示持仓状态（持仓数量 + 当日盈亏），订单数量以小红点或角标提示 | 可见 |

#### 5.2.2 视觉规范
- **位置**：行情页底部固定，不随页面滚动消失
- **高度**：建议 48~52 dp/pt
- **背景**：与行情页主背景区分（如浅灰色卡片 `#F5F5F5` 或带微弱阴影的白色卡片）
- **文字**：
  - 左侧标签：14px，常规色（如 `#333333`）
  - 右侧盈亏金额：16px，加粗，颜色跟随平台统一盈亏色设置（红涨/绿跌或绿涨/红跌）
- **点击热区**：整个条状区域可点击

#### 5.2.3 实时刷新
- 当日盈亏金额与百分比需随行情价格 tick 级联动刷新
- 持仓数量、可卖数量等静态字段在进入行情页时拉取，无需高频刷新

### 5.3 半屏抽屉（Half-Sheet Drawer）

#### 5.3.1 触发方式
- **点击**：点击底部快览条任意区域
- **手势**：在快览条区域**向上滑动**（Swipe Up）展开；在抽屉区域**向下滑动**（Swipe Down）关闭
- **关闭**：点击抽屉外部蒙层、点击关闭按钮、或下滑手势

#### 5.3.2 抽屉布局
- **初始展开高度**：屏幕高度的 55%~65%
- **最大展开高度**：屏幕高度的 85%（支持用户继续上滑至接近全屏）
- **顶部指示器**：抽屉顶部中央提供 36dp 宽的横条（`#DDDDDD`，圆角 2dp），暗示可拖动
- **Tab 切换**：顶部固定 Tab 栏，「持仓」与「订单」两个 Tab

#### 5.3.3 持仓 Tab

**信息卡片布局**：

```
┌─────────────────────────────────────────┐
│  腾讯控股 · 00700                        │
├─────────────────────────────────────────┤
│  持仓数量        可卖数量        成本价   │
│  100 股          100 股         380.00   │
├─────────────────────────────────────────┤
│  当前市值        当日盈亏        持仓盈亏 │
│  67,000.00      +1,234.50     -16,939.25│
│                 (+1.87%)      (-6.64%)  │
├─────────────────────────────────────────┤
│  持仓占比: 11.27%                        │
└─────────────────────────────────────────┘
        [   买入   ]        [   卖出   ]
```

- **字段排列**：采用 3 列网格或上下分组，保证信息密度适中
- **快捷操作**：卡片底部固定「买入」「卖出」两个主按钮，点击后跳转标准交易下单页并**自动带入股票代码**
- **空状态**：若用户从有持仓变为清仓（边界情况），持仓 Tab 显示「暂无持仓」插画与「去交易」按钮

#### 5.3.4 订单 Tab

**列表项布局**（单条未完成订单）：

```
┌─────────────────────────────────────────┐
│  [买入]  限价单          待成交          │
│  委托价: 375.00    委托量: 100          │
│  成交: 0 / 100     09:42:15             │
│                              [撤单]     │
└─────────────────────────────────────────┘
```

- **列表限制**：默认展示该股**最新 10 条**未完成订单，按下单时间倒序排列
- **快捷操作**：
  - 每条订单右侧提供「撤单」按钮，点击后弹出二次确认弹窗，确认后直接调用撤单接口
  - 订单卡片点击（除撤单按钮外）跳转标准交易页的「改单」流程，带入当前订单信息
- **部分成交展示**：若订单状态为「部分成交」，数量展示为「已成交数 / 委托数」，如「30 / 100」
- **空状态**：无未完成订单时显示「暂无进行中订单」与「去下单」按钮

### 5.4 交互流程

#### 5.4.1 正常浏览流程

```
用户进入个股行情页
    ↓
前端异步请求：获取该股持仓信息 + 未完成订单列表
    ↓
根据返回数据判断底部快览条状态（展示/隐藏及内容）
    ↓
用户查看行情，价格变动时实时刷新盈亏字段
    ↓
用户点击/上滑底部快览条 → 展开半屏抽屉
    ↓
用户切换「持仓」/「订单」Tab 查看详情
    ↓
用户点击买入/卖出/撤单/改单 → 跳转对应交易页面
    ↓
用户从交易页返回行情页 → 触发数据刷新
```

#### 5.4.2 撤单交互流程

```
用户在订单 Tab 点击「撤单」
    ↓
弹出二次确认 Dialog：「确定撤销该笔买入委托？」
    ↓
用户确认 → 调用撤单 API
    ↓
API 成功 → 本地移除该订单或更新状态为「已撤单」
    ↓
展示 Toast：「撤单成功」
    ↓
若该笔为最后一条未完成订单，更新底部快览条状态
```

## 6. 状态与边界处理

| 场景 | 处理方式 |
|------|----------|
| 用户未登录 | 底部快览条隐藏，行情页保持现有状态 |
| 已登录，无持仓且无未完成订单 | 底部快览条隐藏 |
| 已登录，有持仓 | 快览条展示持仓数量 + 当日盈亏，盈亏颜色跟随平台统一设置 |
| 已登录，无持仓但有未完成订单 | 快览条展示「进行中订单 · N 笔」|
| 已登录，既有持仓又有订单 | 快览条优先展示持仓信息，订单以角标提示数量 |
| 持仓数据请求失败 | 快览条显示「--」，点击后抽屉内展示错误态与「重试」按钮 |
| 订单数据请求失败 | 抽屉内订单 Tab 展示错误态与「重试」按钮，不影响持仓 Tab |
| 撤单接口失败 | 保留订单在列表中，Toast 提示失败原因（如「已成交，无法撤单」）|
| 行情页跳转下单后返回 | 返回时主动刷新持仓与订单数据 |
| 用户在抽屉内清仓最后一笔持仓 | 关闭抽屉，底部快览条隐藏 |
| 用户在抽屉内撤完最后一笔订单 | 若仍有持仓，订单 Tab 显示空状态；若无持仓，关闭抽屉并隐藏快览条 |

## 7. 接口需求（预估）

### 7.1 获取个股持仓快照
```
GET /api/v1/quotes/{symbol}/position
Request:
  - symbol: 股票代码（如 00700.HK）
  - account_id: 用户账户ID（Header 或 Query）
Response:
  - has_position: bool
  - quantity: int       // 持仓数量
  - sellable_qty: int   // 可卖数量
  - cost_price: decimal // 成本价
  - market_value: decimal
  - daily_pnl_amount: decimal
  - daily_pnl_percent: decimal
  - total_pnl_amount: decimal
  - total_pnl_percent: decimal
  - position_ratio: decimal // 持仓占比
```

### 7.2 获取个股未完成订单
```
GET /api/v1/quotes/{symbol}/orders?status=open&limit=10
Request:
  - symbol: 股票代码
  - status: open（固定查询未完成状态）
  - limit: 10（默认）
Response:
  - orders[]:
    - order_id: string
    - side: BUY | SELL
    - status: PENDING | PARTIAL_FILLED
    - order_type: LIMIT | MARKET | STOP_LIMIT
    - price: decimal
    - quantity: int
    - filled_quantity: int
    - created_at: timestamp
```

### 7.3 撤单接口（复用现有）
```
POST /api/v1/orders/{order_id}/cancel
```

## 8. 非功能性需求

### 8.1 性能
- 底部快览条数据需在行情页首屏加载完成后 **1 秒内**呈现
- 半屏抽屉展开动画时长 **250~300ms**，使用 ease-out 曲线
- 盈亏金额刷新延迟不超过 **500ms**（从行情价格更新到 UI 刷新）

### 8.2 容错与降级
- 若持仓/订单接口任一失败，不影响行情主链路展示
- 撤单操作需处理并发场景（如订单刚好成交），后端返回明确错误码，前端给出友好提示

### 8.3 安全
- 所有持仓与订单接口必须进行身份鉴权
- 撤单操作需校验订单归属，防止越权
